# CRM客户管理系统 - 需求分析文档

## 一、项目背景

### 1.1 项目概述
本项目是一个面向中小型外贸企业的客户关系管理系统（CRM），旨在帮助企业高效管理客户信息、订单数据和销售流程，通过数据统计分析功能辅助决策，提升销售效率和客户满意度。

### 1.2 业务背景
- 外贸企业需要管理大量海外客户信息，包括客户等级、所属国家/大洲、商机信息等
- 需要跟踪订单从下单到付款的全流程，包括采购单号、Lead号、到款金额等
- 需要分析销售数据，了解各区域市场表现、产品销售情况、客户购买行为

### 1.3 项目目标
- 建立统一的客户信息管理平台
- 实现订单全流程跟踪
- 提供多维度数据统计分析
- 支持客户行为深度分析

## 二、功能模块

### 2.1 产品管理模块

#### 功能描述
管理企业销售的产品信息，包括产品名称、型号、单价等基础数据。

#### 功能清单
| 功能 | 描述 |
|------|------|
| 产品列表 | 展示所有产品，支持按名称/型号搜索 |
| 新增产品 | 录入产品名称、型号、单价 |
| 编辑产品 | 修改产品信息 |
| 删除产品 | 移除产品（关联订单的产品需谨慎处理） |

#### 数据字段
- 产品名称（必填）
- 产品型号（必填）
- 单价（必填，美元）

---

### 2.2 客户管理模块

#### 功能描述
管理客户基础信息及联系人，支持多维度筛选查询，与订单数据关联。

#### 功能清单
| 功能 | 描述 |
|------|------|
| 客户列表 | 展示客户信息，支持按公司名/等级/国家筛选 |
| 新增客户 | 录入客户信息及多个联系人 |
| 编辑客户 | 修改客户信息和联系人 |
| 删除客户 | 删除客户（存在关联订单时禁止删除） |
| 跳转订单 | 点击公司名快速查看该客户订单 |
| 客户分析 | 跳转至客户分析详情页 |

#### 数据字段
| 字段 | 说明 |
|------|------|
| 公司名称 | 必填 |
| 客户等级 | A/B/C 三级，用于区分客户重要性 |
| 所属国家 | 客户所在国家 |
| 所属大洲 | 用于区域统计分析 |
| 客户商机 | 记录潜在商业机会 |
| 客户背调 | 客户背景调查信息 |
| 客户性质 | 客户类型描述 |
| 客户来源 | 客户获取渠道 |
| 潜在询价 | 记录潜在订单询价信息 |
| 联系人 | 支持多个联系人（姓名/邮箱/电话） |

---

### 2.3 订单管理模块

#### 功能描述
管理销售订单，包括订单基础信息、关联客户、产品明细等，支持多条件筛选查询。

#### 功能清单
| 功能 | 描述 |
|------|------|
| 订单列表 | 展示订单信息，支持多条件筛选 |
| 新增订单 | 创建订单，选择客户、添加产品明细 |
| 编辑订单 | 修改订单信息和产品明细 |
| 删除订单 | 移除订单及关联的产品明细 |
| 订单筛选 | 按公司名/日期范围/国家/等级/大洲/来源/客户类型筛选 |

#### 数据字段
| 字段 | 说明 |
|------|------|
| 关联客户 | 必填，选择已有客户 |
| 客户类型 | 新客户/老客户 |
| 订单日期 | 下单日期 |
| 付款日期 | 实际收款日期 |
| 采购单号 | Purchase Order编号 |
| Lead号 | 销售线索编号 |
| 到款金额 | 实际收款金额（美元） |
| 发票金额 | Invoice金额 |
| EXW价值 | 工厂交货价 |
| 订单总额 | 订单总金额 |
| 产品明细 | 关联产品、数量、单价 |

---

### 2.4 订单统计模块

#### 功能描述
提供多维度订单数据可视化分析，支持自定义时间范围，帮助管理层了解销售状况。

#### 功能清单
| 功能 | 描述 |
|------|------|
| 全局时间筛选 | 统一设置所有图表的时间范围 |
| 大洲订单分布 | 饼图展示各洲订单数量/金额占比 |
| 到款金额趋势 | 柱状图展示到款金额变化趋势（按月/季度/年） |
| 到款金额对比 | 多年度/月份对比分析 |
| 产品销售排行 | 表格展示产品销售排名（按金额/数量） |
| 产品关联订单 | 点击产品查看购买该产品的所有订单 |

---

### 2.5 客户分析模块

#### 功能描述
深度分析客户购买行为，帮助企业了解客户价值、购买偏好、下单规律。

#### 功能清单
| 功能 | 描述 |
|------|------|
| 客户分析列表 | 按总到款金额排序，展示客户贡献度 |
| 客户搜索 | 按公司名称搜索客户 |
| 下单频率分析 | 展示客户月度下单次数、平均间隔天数 |
| 偏向产品分析 | 展示客户常购产品及购买金额 |
| 下单时间轴 | 时间线展示客户所有订单及产品明细 |

## 五、非功能性需求

### 5.1 性能需求

| 指标 | 要求 |
|------|------|
| 页面加载时间 | 首屏加载 < 3秒 |
| API响应时间 | 普通查询 < 500ms，统计查询 < 2s |
| 并发用户数 | 支持 50+ 并发用户 |
| 数据容量 | 支持 10万+ 订单记录 |

### 5.2 可用性需求

| 指标 | 要求 |
|------|------|
| 系统可用性 | 99% 以上 |
| 故障恢复时间 | < 30分钟 |
| 数据备份 | 支持手动备份 |

### 5.3 安全性需求

| 需求项 | 描述 |
|--------|------|
| 数据安全 | SQLite数据库文件持久化存储，支持备份恢复 |
| 传输安全 | 生产环境建议配置HTTPS |
| 访问控制 | 建议后续版本增加用户认证机制 |
| 数据校验 | 前后端双重数据校验 |

### 5.4 可扩展性需求

| 需求项 | 描述 |
|--------|------|
| 模块化设计 | 前后端采用模块化架构，便于功能扩展 |
| API标准化 | RESTful API设计，便于对接其他系统 |
| 数据库迁移 | 预留数据库迁移接口，可升级至MySQL/PostgreSQL |
| 容器化部署 | Docker容器化，便于横向扩展 |

### 5.5 可维护性需求

| 需求项 | 描述 |
|--------|------|
| 代码规范 | 遵循 ESLint 规范 |
| 注释文档 | 关键业务逻辑需有注释说明 |
| 日志记录 | 服务端记录关键操作日志 |
| 错误处理 | 统一错误处理机制 |
